# Convert project from TypeScript to JavaScript

## Scope

Every source file in `src/` becomes `.jsx` or `.js`. Configs (`vite.config.ts`, `eslint.config.js`, `tsconfig.json`) are reworked or removed. TanStack Router's auto-generated `routeTree.gen.ts` becomes `routeTree.gen.js`.

## Steps

1. **Reconfigure TanStack Router plugin** in `vite.config.ts` → `vite.config.js` to scan `.jsx` route files and emit `routeTree.gen.js`.
2. **Remove `tsconfig.json`** and replace with `jsconfig.json` (keeps the `@/*` path alias working in editors).
3. **Update `eslint.config.js`** to drop `typescript-eslint` rules.
4. **Rename every source file**: `.tsx` → `.jsx`, `.ts` → `.js` across `src/`. ~100 files.
5. **Strip TypeScript syntax** from every file:
   - Remove `interface`, `type` declarations, `as Type` casts, generic parameters `<T>`, return type annotations, parameter type annotations
   - Convert `import type { X }` → remove
   - Rewrite `React.forwardRef<X, Y>` → `React.forwardRef`
   - Rewrite `cva` generics, Zod inferred types, etc.
6. **Update every internal import** that uses `.tsx`/`.ts` extensions (TanStack route splitter, server fns) to `.jsx`/`.js`.
7. **Regenerate** `src/routeTree.gen.js` via dev server boot, fix any route-string mismatches.
8. **Boot dev server, fix runtime errors** until home + each route loads.

## Technical notes / risks

- TanStack Start's Vite plugin and server-fn transformer expect TS source by default. The router plugin accepts a `routeFileIgnorePattern` and we'll configure file extensions explicitly; the server-fn transformer should work on `.js` but is less battle-tested.
- `routeTree.gen.ts` is regenerated by the plugin on dev start — after reconfig it should emit `.js`. If not, we delete it manually and let the plugin re-emit.
- All `*.functions.ts` and `*.server.ts` files (server functions, Supabase auth middleware) lose their type guards. The "client cannot import `.server.ts`" import-protection rule keys off filename — renaming to `.server.js` should still work but needs verification.
- `src/integrations/supabase/types.ts` (generated Supabase types) becomes useless as `.js` — will delete it.
- shadcn UI components use heavy generics (`forwardRef<Ref, Props>`, `VariantProps<typeof cva>`). Mechanically stripping these is straightforward but tedious across ~45 files.
- ESLint may complain loudly until reconfigured; we'll loosen it to JS-only rules.
- After migration, future Lovable tooling, type-aware components, and the strict-build invariant in the system prompt no longer apply — you lose type safety permanently.
